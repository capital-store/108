# Capital Store MSK — сайт (код)

Витрина брендовой одежды в стиле оригинала: чёрная шапка, крупный serif-заголовок,
полноэкранное фото-hero, кремовые кнопки. Плюс три рабочие функции:

- **Корзина** — добавление товара, счётчик, изменение количества, оформление. Работает на фронтенде (сохраняется в браузере).
- **Импорт товаров из VK** — каталог подтягивается из VK Market вашего сообщества.
- **Заявки в Telegram** — оформление заказа уходит сообщением в ваш Telegram-бот.

## Быстрый старт

### Вариант A — только посмотреть дизайн (без сервера)
Откройте `index.html` в браузере. Каталог покажет демо-товары, корзина работает,
формы показывают «спасибо» без реальной отправки. Все файлы держите в одной папке.

### Вариант B — с VK-импортом и Telegram (нужен Node.js 18+)
```bash
cp .env.example .env      # создать конфиг
# заполнить .env токенами (см. ниже)
npm start                 # запустит http://localhost:3000
```
Сервер сам раздаёт статику и отвечает на `/api/products` и `/api/order`.

## Настройка .env

| Переменная | Что это | Где взять |
|------------|---------|-----------|
| `VK_TOKEN` | токен сообщества с доступом к Market | Управление сообществом → Работа с API → Ключи доступа |
| `VK_GROUP_ID` | числовой id сообщества | из `vk.com/club123456` → `123456` |
| `TG_BOT_TOKEN` | токен бота | создать бота у [@BotFather](https://t.me/BotFather) |
| `TG_CHAT_ID` | куда слать заявки | узнать свой id у [@userinfobot](https://t.me/userinfobot); для группы — добавьте туда бота |

Если VK не настроен — сайт покажет демо-каталог. Если Telegram не настроен — заявка не уйдёт (в консоли будет ошибка).

## Структура

```
index.html      Главная (фото-hero, категории, хиты, отзывы, FAQ, подписка)
catalog.html    Наличие — каталог с фильтром по категориям + корзина
logistics.html  Логистика — калькулятор доставки, тарифы, заявка
support.html    Поддержка — форма вопроса
contacts.html   Контакты — ссылки на мессенджеры
blog.html       Новости
styles.css      Все стили (палитра оригинала)
app.js          Каталог, корзина, оформление, калькулятор
partials.js     Шапка, футер, корзина-дровер, модалка (общие для всех страниц)
server.js       Backend: VK-импорт + Telegram + раздача статики
```

## Как работает импорт из VK

`server.js` вызывает метод `market.get` VK API, приводит товары к единому виду
(id, название, цена, старая цена, категория, фото, ссылка) и кэширует на 5 минут.
Категория угадывается по названию — при желании поправьте функцию `guessCat`.
Фронтенд запрашивает `/api/products`; при недоступности — берёт демо-каталог из `app.js`.

## Как работают заявки в Telegram

Оформление заказа (или заявка на товар) отправляется `POST /api/order`.
Сервер формирует сообщение (имя, телефон, состав корзины, сумма) и шлёт его
через `sendMessage` в ваш чат/бот. Формат сообщения — в функции `formatOrder`.

## Деплой

Подойдёт любой хостинг с Node.js (VPS, Render, Railway) — команда `npm start`.
Для бесплатного варианта на Vercel эндпоинты `/api/*` можно переписать в serverless-функции;
статику Vercel раздаст сам. Токены задаются в переменных окружения панели хостинга (не в коде!).

## Важно про безопасность

Токены VK и Telegram **никогда не должны попадать во фронтенд** (в `.js`, который грузит браузер).
Они живут только в `.env` на сервере. Поэтому импорт и отправка идут через backend, а не напрямую из браузера.

## Что доработать под прод

- Заменить URL фото-hero в `index.html` на своё изображение.
- Проставить бренды/категории товарам в VK (или доработать `guessCat`).
- Подключить аналитику (Яндекс.Метрика), sitemap, HTTPS.
- Настроить антиспам на форму (honeypot уже можно добавить в модалку).
