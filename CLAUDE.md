# CLAUDE.md — контекст проекта для Claude Code

Этот файл читает Claude Code при старте. Здесь — карта проекта, договорённости и
список задач, чтобы можно было продолжать разработку без повторного изучения кода.

## Что это

Сайт магазина **Capital Store MSK** — витрина оригинальной брендовой одежды с
доставкой из-за рубежа. Пересобран кодом с сайта на Tilda (capitalstoremsk.ru).
Модель: клиент смотрит товар → добавляет в корзину / оставляет заявку → заявка
уходит менеджеру в Telegram. Классической онлайн-оплаты нет.

## Стек

- **Фронтенд:** статические HTML + один общий `styles.css` + ванильный JS
  (`app.js`, `partials.js`). Без сборщика и фреймворка — специально, чтобы легко
  хостить и дорабатывать. Шрифты: Playfair Display (serif, заголовки) + Inter (текст).
- **Бэкенд:** `server.js` — чистый Node.js (только встроенные модули `http/https/fs`),
  без Express и без npm-зависимостей в рантайме. Раздаёт статику + два API-эндпоинта.
- **Хранилище:** товары приходят из VK Market; корзина — в `localStorage` браузера.
  Своей БД нет.

## Структура файлов


## Как работают ключевые части

### Общие блоки (partials.js)
Каждая страница содержит placeholder-ы:
`<div id="site-header" data-page="...">`, `<div id="site-footer">`,
`<div id="site-overlays">`. `partials.js` синхронно заменяет их на реальную
разметку шапки/футера/корзины/модалки. Активный пункт меню задаётся через
`data-page` (значения: catalog, logistics, support, contacts, blog).
Порядок подключения важен: сначала `app.js`, потом `partials.js`.

### Данные о товарах (app.js)
`loadProducts()` делает `fetch('/api/products')`. Если backend недоступен —
использует `FALLBACK_PRODUCTS` (демо). Модель товара:
```
{ id, name, brand, price, old, cat, img, images[], desc, sizes[], tag }
```
`cat` — один из id категорий: outer, pants, hoodie, tshirt, bags, shoes, acc.

### Корзина (объект Cart в app.js)
Хранится в `localStorage` (ключ `capital_cart`). Разные размеры одного товара —
отдельные строки (ключ `id::size`). Рендерится в выезжающий дровер. Оформление
открывает модалку → `POST /api/order`.

### Backend (server.js)
- `GET /api/products` → VK `market.get`, нормализует и кэширует на 5 минут.
  Если `VK_TOKEN`/`VK_GROUP_ID` не заданы — возвращает `[]` (фронт покажет демо).
- `POST /api/order` → шлёт заявку в Telegram (`sendMessage`). Тело: `{name, phone,
  email, delivery, item, cart:{items,total}}`.
- Всё остальное — статика из корня проекта.

## Запуск

```bash
cp .env.example .env      # заполнить токены
npm start                 # http://localhost:3000
```
Без сервера: открыть `index.html` в браузере (демо-каталог, корзина работает,
заявки не отправляются).

## Договорённости по коду

- Без внешних runtime-зависимостей на бэкенде (только стандартная библиотека Node).
- Фронтенд — без сборки; новые страницы копируют структуру существующих
  (placeholder-ы + подключение `app.js`/`partials.js`).
- Цвета — только через CSS-переменные из `:root`, не хардкодить hex по месту.
- Секреты (VK/TG токены) — только в `.env` на сервере, никогда во фронтенд-JS.
- Тексты — на русском.

## Проверка (как тестировали)

Фронтенд проверялся через jsdom: инъекция шапки/футера/корзины, рендер каталога и
хитов, добавление/изменение/удаление в корзине, страница товара с выбором размера.
Бэкенд — запуск в демо-режиме: `/api/products` → `[]`, `/api/order` без токенов
возвращает понятную ошибку, статика отдаётся 200. При доработке — прогонять те же
сценарии вручную в браузере или дописать тесты.
